Eres un modelo de lenguaje asistiendo en el desarrollo y mantenimiento del siguiente proyecto:

Nombre: Portfolio Daniel Tunjano
Tipo: Portafolio personal profesional
Repositorio: portfolio-daniel_tunjano

Características principales:

Sitio web estático y responsivo para mostrar proyectos, experiencia y contacto.
Multilenguaje (español/inglés) usando archivos de internacionalización.
Optimización SEO y performance.
Integración de datos de proyectos desde archivos Markdown y JSON.
Uso de componentes reutilizables y diseño moderno.
Estructura del proyecto:

src/: Código fuente principal.
components/: Componentes Astro y React (algunos en TypeScript).
content/: Configuración y contenido de proyectos (Markdown y TypeScript).
data/: Archivos JSON con datos de proyectos y SEO.
i18n/: Archivos de internacionalización.
layouts/: Layouts base en Astro.
lib/: Utilidades y lógica de negocio (TypeScript).
pages/: Páginas del sitio (Astro, rutas dinámicas y estáticas).
scripts/: Scripts de automatización (TypeScript).
styles/: Estilos globales (Tailwind CSS).
utils/: Utilidades varias (JS/TS).
public/: Recursos estáticos (imágenes, fuentes).
Stack de tecnologías:

Astro (framework principal, archivos .astro)
React (componentes específicos, .tsx)
TypeScript y JavaScript
Tailwind CSS (estilizado)
Markdown (contenido de proyectos)
pnpm (gestor de paquetes)
Notion API (integración opcional)
Herramientas de SEO y optimización
Herramientas y dependencias clave:

pnpm para gestión de dependencias y scripts.
Tailwind CSS para estilos utilitarios.
Astro para generación de sitios estáticos y SSR.
React para componentes interactivos.
Markdown para contenido estructurado.
Notion API para obtención de datos (si aplica).
Scripts personalizados en scripts/ para automatización.
Configuración en archivos como astro.config.mjs, tailwind.config.js, tsconfig.json.
Consideraciones:

El código fuente está en src/.
Los datos de proyectos y SEO están en data/ y content/project/.
Los estilos globales están en styles/global.css.
El sitio soporta internacionalización desde src/i18n/.
El build y desarrollo se gestiona con pnpm y scripts definidos en package.json.
Por favor, ten en cuenta esta estructura y stack al analizar, modificar o generar código para este proyecto.
